# 現状の課題

更新日: 2026-03-01

## 1. `make test` が正常完了しない
- 内容:
  `Makefile` の `test-frontend` が `npm test` を呼び出すが、`frontend/package.json` に `test` スクリプトが定義されていない。
- 影響:
  ルートの `make test` でフロントエンドのテスト工程が必ず失敗する。
- 根拠:
  - `Makefile` の `test-frontend`: `cd frontend && npm test`
  - `frontend/package.json` の `scripts` に `test` が存在しない

## 2. API テスト実行が環境依存で失敗しやすい
- 内容:
  `api` の `go test ./...` が、環境によっては Go のキャッシュディレクトリ権限で失敗する。
- 影響:
  CI/実行環境差異でテストが不安定になり、検証の再現性が落ちる。
- 根拠:
  - `go test ./...` 実行時に `~/Library/Caches/go-build` の `operation not permitted` が発生する環境がある
  - `GOCACHE=/tmp/go-build-cache` を指定するとテストは成功

## 3. 中核ユースケース (`generate_plan`) の自動テストが未整備
- 内容:
  旅程生成の主処理 `api/internal/usecase/generate_plan.go` に対するテストファイルがない。
- 影響:
  外部依存（Places/LLM/Repository）を含む変換ロジックの回帰検知ができない。
- 根拠:
  - `go test ./...` 出力で `api/internal/usecase` は `[no test files]`

## 4. ドキュメントがテンプレート状態のまま
- 内容:
  `frontend/README.md` が Vite の初期テンプレート内容のままで、プロジェクト固有の手順・設計情報が不足している。
- 影響:
  新規参加者がセットアップ/実行方法を誤りやすく、運用コストが上がる。
- 根拠:
  - `frontend/README.md` が React+Vite テンプレート説明中心で、Travelist 固有情報がほぼない

## 6. 快適なUI・UXにするための指標（KPI）

### 6-1. 体感速度（Performance）
- 指標:
  初回表示時間（LCP）、画面操作の応答速度（INP）、レイアウト崩れ（CLS）
- 目標値:
  - LCP: 2.5秒以下（モバイル4G想定）
  - INP: 200ms以下
  - CLS: 0.1以下

### 6-2. プラン作成の完了しやすさ（Task Completion）
- 指標:
  「条件入力→旅程生成」完了率、完了までの平均時間、離脱率
- 目標値:
  - 完了率: 80%以上
  - 完了までの時間: 3分以内
  - フォーム離脱率: 20%未満

### 6-3. 入力しやすさ・エラー耐性（Form UX）
- 指標:
  バリデーションエラー発生率、エラー復帰率（エラー後に完了できた割合）
- 目標値:
  - エラー発生率: 15%未満
  - エラー復帰率: 90%以上

### 6-4. 旅程の理解しやすさ（Readability）
- 指標:
  旅程ページでの主要情報到達時間（当日予定/移動時間/予算）
- 目標値:
  - 主要情報到達時間: 10秒以内

### 6-5. 編集のしやすさ（Plan Editability）
- 指標:
  旅程編集機能の利用率、編集後保存率、編集操作の取り消し率
- 目標値:
  - 編集後保存率: 85%以上
  - 取り消し率: 10%未満

### 6-6. 継続利用（Retention）
- 指標:
  7日後再訪率、30日後再訪率、保存機能利用率
- 目標値:
  - 7日後再訪率: 25%以上
  - 30日後再訪率: 10%以上
  - 保存機能利用率: 60%以上

### 6-7. ユーザー満足度（Satisfaction）
- 指標:
  CSAT（満足度）、NPS、生成旅程に対する「そのまま使える」回答率
- 目標値:
  - CSAT: 4.2/5 以上
  - NPS: +20 以上
  - 「そのまま使える」回答率: 70%以上
